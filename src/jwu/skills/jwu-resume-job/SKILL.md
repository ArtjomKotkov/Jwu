# jwu: подхват работы (job) после потери контекста

## Overview

Контекст очистился / новая сессия / вернулись к задаче через время — а состояние работы ушло.
Прежде чем продолжать или писать код, **восстанови состояние из jwu** (локальная БД, без сети) и
кратко синхронизируйся с пользователем. Не начинай «с чистого листа» по задаче, у которой уже есть работа.

## Когда

- После `/clear`, компакта контекста или в начале новой сессии по уже идущей задаче.
- Пользователь говорит «продолжим / на чём остановились / что по <KEY>».
- Ты собираешься залогировать прогресс (jwu-track-job), но лога работы в контексте нет.
- **Не для старта новой задачи** — это jwu-start-job. **Не для нового цикла правок** — там новая работа.

## Шаги

1. **Найти работу.**
   - Знаешь ключ задачи: `jwu jobs --task <KEY> --json` (бери активную).
   - Не знаешь: `jwu jobs --status active --json`. Несколько активных → покажи список
     (`id / task_key / updated_at`) и **спроси, какую**; одна — бери её.

2. **Загрузить лог:** `jwu job show <JOB_ID>` (или `--json` для разбора по типам).

3. **Реконструировать «где мы»** по типам записей:

   | Что ищешь | Тип записи | Зачем |
   |---|---|---|
   | текущая фаза | последняя `phase` (+ `--status`) | на каком этапе процесса |
   | что осталось | `todo` без `status=done` | незакрытые хвосты |
   | незакрытые баги | `bug` без парного `bug-resolved` | что ещё чинить |
   | действующие запреты | `constraint` (⛔) | чего делать НЕЛЬЗЯ — НЕ нарушай |
   | принятые решения | `decision` (🧭) | не пересматривай без причины |
   | состояние тестов | последний `test-pass`/`test-fail` | зелено/красно сейчас |

4. **Синхронизируйся с пользователем** — короткая сводка: задача, текущая фаза, что сделано,
   что осталось, действующие запреты. Подтверди следующий шаг, прежде чем писать код.

5. **Дальше веди работу** через jwu-track-job (фиксируй фазы/баги/тесты/решения).

## Частые ошибки

- Начать заново и завести **новую** работу, хотя по задаче уже есть активная → проверь `jwu jobs --task <KEY>` ПЕРЕД стартом.
- Пропустить `constraint`-записи и нарушить запрет (напр. «код правим только локально»).
- Продолжать по памяти, не перечитав `jwu job show` → расхождение с реальным состоянием.

## Подтверждение правок

Подхват и чтение — без подтверждения. Но **правки КОДА, коммиты и пуш — только после явного
подтверждения пользователя** (правило проекта). Старт/анализ — в jwu-start-job, ведение — в jwu-track-job.
