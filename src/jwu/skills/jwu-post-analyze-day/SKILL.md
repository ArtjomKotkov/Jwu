# jwu: итоги дня по работам (для трекинга времени)

## Overview

Пройтись по всем jwu-работам (jobs), затронутым **сегодня**, и для каждой задачи дать
**очень короткую** выжимку «что сделано» — чтобы пользователь разнёс время по задачам.
Это ретроспектива (что уже сделано), а не план на будущее.

Источник — локальная БД jwu, без сети: одна команда отдаёт работы со всеми записями.

## Шаги

1. Собери работы за сегодня (записи с сегодняшней датой). Дату можно передать аргументом
   (`/jwu-post-analyze-day 2026-05-21`), иначе берётся сегодняшняя:

   ```bash
   today=${1:-$(date +%F)}
   jwu jobs --json | jq --arg d "$today" '
     map(select(any(.records[]?; .ts[0:10] == $d) or (.updated_at[0:10] == $d)))
     | map({task_key, status,
            prs: [.prs[]? | "\(.repo)#\(.pr_id)"],
            done: [.records[]? | select(.ts[0:10] == $d) | "[\(.kind)] \(.text)"]})'
   ```

   В `done` лежат только сегодняшние записи: `phase` — этапы, `remark` — замечания ревью,
   `note` — заметки/результаты. Если массив пуст — за день по работам ничего нет, так и скажи.

2. Сожми каждую задачу в **5-10 слов**. Если сделано много — несколько коротких фраз
   (по 5-8 слов), а не пересказ записей. Не копируй текст записей дословно.

   **Обобщай рутину без деталей:**
   - тесты — пиши просто «провёл тесты» (без числа passed, имён файлов, окружения);
   - конфликты/мердж — просто «разрешил конфликт» (без числа коммитов/конфликтов,
     списка файлов, рецепта);
   - оставляй только суть: что по задаче реально сделано/решено.

3. Выведи список: **одна задача — одна строка, только номер задачи**. Без PR,
   без `repo#id`, без ссылок и скобок с метаданными:

   ```
   PROJ-12593: перенёс фикс stat v2 в release-10.8, провёл тесты
   PROJ-399: разрешил конфликт, откатил low-memory конфиг, ответил ревьюерам
   ```

   Несколько работ по одной задаче — объедини в одну строку.

## Заметки

- Read-only: только чтение БД jwu, ничего не пишем и не синкаем.
- `ts`/`updated_at` — UTC ISO; фильтр по `[0:10]` сравнивает дату. Работал за полночь —
  передай нужную дату аргументом.
- Это итог дня. «Что делать дальше / на ревью / поправить» — это `/jwu-analyze-day`.
